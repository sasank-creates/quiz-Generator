 The AI-Powered Real-Time Interview Proctoring System is a real-time
proctoring platform designed to ensure fairness and integrity during remote
online interviews. The system leverages artificial intelligence, computer vision,
and modern full-stack web technologies to detect and prevent cheating
behaviors such as tab switching, gaze deviation, multiple face detection, mobile
phone usage, and use of unauthorized materials.
 The platform is built using a modern technology stack consisting of React with
TypeScript for the frontend, Node.js with Express.js for the backend, and MongoDB
for persistent data storage. Real-time communication between the candidate's
browser and the monitoring server is achieved through Socket.IO, enabling instant
event transmission and score updates.
 On the client side, TensorFlow.js with the BlazeFace model performs edge-
AI face detection directly in the browser, identifying multiple faces without
sending every frame to the server. A pixel-difference frame filtering algorithm
reduces unnecessary network traffic by only transmitting frames with significant
visual changes. On the server side, OpenAI's GPT-4o-mini vision model analyzes
sampled video frames for advanced cheating indicators such as phone usage,
unauthorized notes, and multiple persons.
 The system implements a trust score mechanism starting at 100, which
decreases based on detected violations using an exponential decay penalty model. A
heartbeat-based liveness system with challenge-response verification ensures the
candidate remains present and the connection is not tampered with. An interactive
dashboard provides real-time visualization of trust scores, event logs, and session
statistics for proctors.

1.1 Introduction to AI Interview Proctoring
 The AI-Powered Real-Time Interview Proctoring System is designed to provide
organizations and educational institutions with a robust, real-time proctoring solution
that ensures fairness and integrity during remote online interviews and examinations.
In the context of modern hiring processes and online assessments, where candidates
participate from diverse and uncontrolled environments, the need for automated
cheating detection has become critical. This system addresses that need by
combining artificial intelligence, computer vision, and real-time web technologies into
a cohesive, scalable platform.
 The project employs TensorFlow.js with the BlazeFace model for edge-AI face
detection directly in the candidate's browser, eliminating the need to send every
video frame to the server and significantly reducing latency and bandwidth usage. On
the server side, OpenAI's GPT-4o-mini vision model performs advanced analysis of
sampled video frames to detect sophisticated cheating indicators such as mobile
phone usage, unauthorized notes, and the presence of multiple persons. The entire
system is built using a modern MERN-inspired architecture with React (TypeScript)
on the frontend, Node.js with Express.js on the backend, and MongoDB for data
persistence. Real-time communication is achieved through Socket.IO, enabling
instant event transmission, score updates, and challenge-response verification
between the client and server.
 In most organizations and educational institutions, conducting fair remote
interviews and online assessments remains a significant challenge. Traditional
proctoring methods rely on human invigilators who must physically monitor each
candidate, making them time-consuming, labor-intensive, and impractical for large-
scale remote assessments. Furthermore, candidates attempting to cheat use
increasingly sophisticated methods — using secondary devices, having other
persons present, referring to unauthorized materials, or switching browser tabs —
that are difficult for human proctors to detect consistently.
1
 The main objective of the AI-Powered Real-Time Interview Proctoring System
is to develop a real-time, intelligent proctoring platform that automatically detects
and penalizes cheating behaviors during remote interviews and online
assessments. The system uses TensorFlow.js with BlazeFace for client-side face
detection and OpenAI's GPT-4o-mini for server-side advanced vision analysis to
identify suspicious activities including multiple faces, mobile phone usage,
unauthorized notes, gaze deviation, and tab switching.
 To enhance reliability, the platform implements a heartbeat-based liveness
system with challenge-response verification, ensuring the candidate's presence
and preventing connection tampering. A pixel-difference frame filtering algorithm
optimizes network traffic by only transmitting frames with significant visual
changes, reducing bandwidth consumption by up to 85%.
 The trust score engine employs an exponential decay penalty model where
repeated violations incur progressively heavier penalties, accurately reflecting
escalating risk. Built on the MERN stack with Socket.IO for real-time
communication, the system ensures instant event processing and live dashboard
updates. An interactive proctor dashboard provides real-time visualization of trust
scores, event logs, and session statistics, enabling proctors to monitor multiple
candidates simultaneously.
Overall, the project integrates edge AI, cloud AI, real-time communication, and
adaptive scoring to deliver an efficient, secure, and fair proctoring
experience.

 Additionally, while certain existing online proctoring solutions attempt to

proctoring platform that combines edge computing (TensorFlow.js), cloud AI vision

reliable, fair, and scalable interview proctoring accessible from any web browser.

1.6 Methodology Overview
4
 The AI-Powered Real-Time Interview Proctoring System follows a systematic
and modular methodology to ensure accuracy, efficiency, and scalability. The
process begins with session initialization, where a new proctoring session is
created in MongoDB and the candidate's browser establishes a Socket.IO
connection with the backend server.
 The candidate's webcam feed is captured using the browser's MediaDevices API
and processed locally through two parallel pipelines: (1) TensorFlow.js with
BlazeFace performs real-time face detection to identify multiple faces without
server round-trips, and (2) a pixel-difference algorithm compares consecutive
frames to determine if significant visual change has occurred before transmitting
the frame to the server.
 On the server side, the monitorSocket module receives video frames and
processes them through OpenAI's GPT-4o-mini vision model using structured
JSON output to detect cheating indicators. The scoring service applies
exponential decay penalties to the trust score based on violation type and
frequency. Simultaneously, a heartbeat-based liveness system monitors the
connection, applying penalties for socket tampering, while random challenge-
response prompts verify the candidate's physical presence.
 The frontend provides two views: a Candidate view with live camera feed,
session status, and challenge notifications, and a Dashboard view with real-time
trust score cards, event logs, and session statistics. The entire system
communicates via Socket.IO events, ensuring sub-second latency for all
monitoring and feedback operations.

4.2 TESTING METHODOLOGY
 The Testing and Validation phase ensures that the AI-Powered Real-Time
Interview Proctoring System functions accurately, efficiently, and reliably across all
its components. This stage verifies that the edge-AI face detection model correctly
identifies multiple faces, the server-side OpenAI vision analysis accurately detects
cheating indicators, the trust score engine applies correct exponential decay
penalties, the Socket.IO real-time communication delivers events with minimal
latency, and the heartbeat-based liveness system correctly identifies connection
tampering. The goal of this phase is to confirm that the system delivers a robust,
fair, and responsive proctoring experience without excessive false positives or
missed detections.
 The testing process followed a structured approach involving multiple test
cases and real-world scenarios. Each component of the system was tested
independently (unit testing) and then as an integrated whole (integration testing).
The testing methodology included the following steps:
1. Edge-AI Face Detection Test: The BlazeFace model was tested with various
scenarios including single face, multiple faces at different distances and angles, no
face present, and partially occluded faces. The model was evaluated for detection
accuracy, false positive rate, and processing latency in the browser environment.
2. OpenAI Vision Analysis Test: The GPT-4o-mini vision model was tested with
curated frames containing phones, notebooks, multiple people, and clean frames. The
structured JSON output was verified for consistency and correctness across all test
cases.
3. Pixel-Difference Frame Filtering Test: The pixel-difference algorithm was
tested with static camera feeds (should not send frames), normal movements (should
occasionally send frames), and rapid movements (should consistently send frames).
The threshold of 15% pixel change was validated across different lighting conditions.
4. Trust Score Engine Test: The exponential decay penalty model was verified
mathematically. Test cases included single violations, repeated violations of the same
type, and mixed violation types to ensure correct penalty accumulation and score
floor at zero.

5.1 INTRODUCTION
5.2 SYSTEM IMPLEMENTATION
The implementation follows a modular architecture with clear separation of
concerns:
The implementation of the AI-Powered Real-Time Interview Proctoring System
integrates edge AI, cloud AI, and modern full-stack web technologies to create a
real-time, intelligent proctoring platform. The system combines a client-side
TensorFlow.js face detection model with server-side OpenAI GPT-4o-mini vision
analysis, connected through
a Socket.IO real-time communication layer. Video frames captured from the
candidate's webcam are processed through a pixel-difference filter before being
analyzed, and the results drive an exponential decay trust score engine. The backend
uses Node.js, Express.js, and MongoDB, while the React TypeScript frontend provides
interactive candidate and proctor views.
A. Backend Implementation: The backend server is built with Node.js and
Express.js, using MongoDB for persistent storage. The server.js entry point creates an
HTTP server, establishes a Socket.IO connection with CORS support, and connects to
MongoDB with retry logic. The monitorSocket module handles all real-time
communication including session management, heartbeat monitoring, video frame
processing, and event broadcasting. The session model stores the userId, start time,

CONCLUSION AND FUTURE SCOPE
6.1 CONCLUSION
 The project "AI-Powered Real-Time Interview Proctoring System" provides an
efficient and intelligent solution for ensuring fairness and integrity during remote
online interviews and assessments. By combining edge AI (TensorFlow.js with
BlazeFace) for client-side face detection with cloud AI (OpenAI GPT-4o-mini) for
server-side advanced vision analysis, the system achieves a hybrid detection
architecture that balances accuracy, latency, and cost.
 The integration of TensorFlow.js BlazeFace for browser-based face detection
ensures immediate detection of multiple faces without server round-trips, achieving
sub-100ms latency. The pixel-difference frame filtering algorithm reduces
unnecessary network traffic by approximately 85%, and the 1/10 frame sampling
strategy for OpenAI API calls reduces cloud AI costs by up to 90%. The structured
JSON output format with strict schema validation ensures reliable, parseable
responses from the AI vision model.
 The exponential decay penalty model implemented in the scoring service
provides a fair and graduated response to cheating violations, where repeated
offenses incur progressively heavier penalties. The heartbeat-based liveness
system with 5-second timeout detection and random challenge-response prompts
(LOOK_LEFT, LOOK_RIGHT, SMILE) ensures continuous verification of the
candidate's physical presence and prevents connection tampering.
 The application includes a comprehensive proctor dashboard with real-time trust
score visualization, categorized event logging with severity classification, and session
statistics. The candidate view provides a clean, non-intrusive monitoring interface
with challenge notifications and session status indicators.
 Overall, the project demonstrates the effective integration of edge AI, cloud AI,
real-time WebSocket communication, and adaptive scoring algorithms to deliver a
robust, scalable, and fair proctoring platform. It serves as a production-ready
foundation for organizations seeking to implement automated interview integrity
monitoring in their remote hiring and assessment workflows.

6.2 FUTURE SCOPE
While the current implementation provides a comprehensive real-time proctoring
solution, several enhancements can be explored in future iterations:
1. Audio Analysis Integration: Incorporating audio analysis to detect suspicious sounds
such as whispering, dictation from another person, or background conversations using
the Web Audio API.
2. Screen Recording and Analysis: Adding browser screen capture capabilities
using the Screen Capture API to monitor what the candidate is viewing on their
screen.
3. Multi-Session Proctor Dashboard: Extending the dashboard to support
monitoring multiple candidates simultaneously with a grid view of video feeds and
priority-based alerting.
4. Persistent Session History and Analytics: Implementing a comprehensive
session history system with detailed reports, playback at flagged timestamps, and
aggregate analytics.
5. Mobile Browser Optimization: Optimizing the edge-AI pipeline for mobile
browsers with reduced model sizes and adaptive frame rates based on device
capabilities.
6. Custom AI Model Training: Training custom object detection models (e.g.,
YOLO or EfficientDet) specifically for the proctoring domain, reducing dependency
on third-party AI APIs.
7. Integration APIs: Developing RESTful and webhook-based integration APIs to
allow seamless embedding into existing interview platforms and LMS systems.
8. Accessibility Features: Implementing screen reader support, keyboard
navigation, high-contrast modes, and configurable alert modalities for candidates
with disabilities.
